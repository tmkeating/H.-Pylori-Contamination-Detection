## 🔬 Project: H. Pylori Contamination Detection
**Objective:** Detect *H. pylori* bacteria in histology slides.
**Constraint:** Achieve clinical-grade throughput (>500 img/s) and break the 92% patient accuracy bottleneck.

### 🛠️ Technical Stack (Iteration 21: Multi-Backbone Ensemble Calibration)
- **Architecture**: **ConvNeXt-Tiny** (Global Morphology) & **ResNet50** (Local Patterns) + **Gated Attention MIL**.
- **Bag Strategy**: **Dynamic 500-Patch CPU Loading** + **Memory-Efficient Gradient Checkpointing**.
- **Centralized Control**: **profiles.sh** (Single Source of Truth for AUDITOR/SEARCHER hyper-profiles).
- **Optimization Strategy**: **AdamW** (LR=2e-5, WD=0.05-0.1) & **SWA** (Stochastic Weight Averaging) at `swa_lr=1e-5`.
- **Loss Function**: **Asymmetric Focal Loss** with Dynamic Weighting ($PosWeight=2.2$ for Searcher stability).
- **Metric Tracking**: **F1-Score Model Checkpointing** to prevent Searcher class collapse.
- **Hardware Optimization**: **A40-Native FP32 Matmul ('high')** + **Local SSD Scratch Syncing**.

## 📈 Current Performance & Milestones
1. **The 0.05 Wall**: Identified a systematic feature-representation failure where sparse infections ("Ghost Patients") hover at $P \approx 0.05$.
2. **Strategy Pivot**: Transitioned from a single-backbone model to a **Diversity Ensemble** (ConvNeXt + ResNet) to capture different bacterial textures (7x7 vs 3x3 kernels).

4. **ResNet Stability (Iter 21)**: Resolved "Double-Weighting" collapse by disabling `WeightedRandomSampler` for weighted runs and calibrating $PosWeight$ to 2.2.
